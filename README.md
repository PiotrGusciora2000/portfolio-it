# Konfiguracja i optymalizacja platformy PC (Intel Z270 + i7-6700K)

## Skrócony opis projektu

Modernizacja platformy PC poprzez wymianę CPU (i5-7600K → i7-6700K), konfigurację BIOS/UEFI oraz overclocking i undervolting procesora. Zdiagnozowano BSOD `CLOCK_WATCHDOG_TIMEOUT` spowodowany zbyt niskim napięciem CPU i ustabilizowano system przy 4.4 GHz @ 1.23–1.24V. Wykonano testy OCCT (1h bez błędów) oraz optymalizację Windows 10 przy użyciu PowerShell GUI (Chris Titus Tech Utility).

# Problem / objawy

Po wymianie procesora z Intel i5-7600K na Intel i7-6700K występowały:

- błędy systemowe typu BSOD:
  - `CLOCK_WATCHDOG_TIMEOUT`
- niestabilność systemu pod obciążeniem przy agresywnym undervoltingu
- wysokie temperatury CPU
- zwiększona praca wentylatorów przy standardowym napięciu
- spadki FPS oraz niestabilny frametime w grach CPU-zależnych

---

# Diagnostyka / analiza

Przeprowadzono analizę stabilności platformy po wymianie CPU i konfiguracji BIOS.

Zakres diagnostyki:

- testy różnych wartości napięcia Vcore przy OC 4.4–4.5 GHz
- analiza zależności między:
  - napięciem CPU,
  - temperaturą,
  - stabilnością systemu,
  - występowaniem BSOD
- identyfikacja przyczyny błędu `CLOCK_WATCHDOG_TIMEOUT`
- testy stabilności przy użyciu OCCT
- analiza bottleneck CPU na podstawie zachowania FPS i obciążenia systemu

---

# Wykonane działania

- Wymiana CPU: Intel i5-7600K → Intel i7-6700K
- Reset CMOS po wymianie procesora

## Konfiguracja BIOS/UEFI

- konfiguracja mnożnika CPU
- konfiguracja napięcia Vcore
- konfiguracja Load-Line Calibration (LLC)
- aktywacja profilu XMP

## Overclocking / undervolting

- overclocking procesora do zakresu 4.4–4.5 GHz
- undervolting przy taktowaniu 4.4 GHz:
  - redukcja napięcia CPU
  - optymalizacja temperatur
  - ograniczenie poboru energii
  - zmniejszenie intensywności pracy wentylatorów

## Stabilizacja systemu

- korekta napięcia po wykryciu niestabilności systemu
- długotrwałe testy stabilności po każdej zmianie parametrów

## Optymalizacja Windows 10

Optymalizacja systemu przy użyciu Chris Titus Tech Utility (PowerShell GUI):

- wyłączenie zbędnych usług działających w tle
- ograniczenie telemetrii
- konfiguracja ustawień prywatności
- redukcja obciążenia systemu

# Rozwiązanie / efekt

Uzyskano stabilną konfigurację:

- 4.4 GHz @ 1.23–1.24V

Wyniki:

- OCCT – 1h bez błędów
- eliminacja BSOD `CLOCK_WATCHDOG_TIMEOUT`
- obniżenie temperatur CPU pod obciążeniem
- zmniejszenie poboru energii dzięki undervoltingowi
- poprawa kultury pracy chłodzenia

# Wnioski techniczne

- Zbyt agresywny undervolting może powodować błędy synchronizacji rdzeni CPU (`CLOCK_WATCHDOG_TIMEOUT`)
- Fine-tuning napięcia Vcore wpływa bezpośrednio na:
  - stabilność,
  - temperatury,
  - pobór energii,
  - kulturę pracy chłodzenia
- Overclocking wymaga zachowania równowagi między taktowaniem a napięciem CPU
- Odpowiednia konfiguracja BIOS i LLC wpływa na stabilność napięcia pod obciążeniem
- Ograniczenie zbędnych procesów Windows może poprawić responsywność systemu i zmniejszyć obciążenie CPU

# Problem / objawy

- Brak uporządkowanego systemu identyfikacji części i dokumentów
- Ręczne wyszukiwanie danych w systemach magazynowych i fakturach
- Problemy z lokalizacją części oraz przypisaniem do konkretnych maszyn
- Chaotyczny obieg dokumentów i ryzyko utraty faktur
- Częste awarie wózków widłowych i brak kontroli nad historią serwisową
- Problemy z dostępem do danych sieciowych (Wi-Fi) w Windows 10
- Brak automatyzacji segregacji wiadomości e-mail
- Nieefektywna obsługa reklamacji technicznych i dostaw części

---

# Diagnostyka / analiza

Przeprowadzono analizę działania systemu magazynowego oraz przepływu informacji między działami.

Zakres analizy obejmował:

- kompatybilność kodów kreskowych między systemami
- ręczne przepisywanie identyfikatorów i ryzyko błędów
- analizę parametrów technicznych części:
  - średnice,
  - gwinty,
  - rozstaw otworów
- identyfikację problemów w obiegu dokumentów
- analizę przepływu wiadomości e-mail
- weryfikację uszkodzeń dostarczonych pomp oraz odpowiedzialności dostawcy
- obserwację awaryjności wózków widłowych pracujących poza środowiskiem nominalnym

# Wykonane działania

## Systemy i infrastruktura

- Uzyskanie dostępu do zapisanych danych Wi-Fi w Windows 10 przy użyciu CMD
- Rozwiązywanie problemów konfiguracyjnych na podstawie dokumentacji technicznej
- Modyfikacja ustawień programu magazynowego (.app)
- Integracja telefonu z komputerem do zdalnej obsługi stanowiska

## System identyfikacji części

Opracowanie systemu identyfikacji części obejmującego:

- generowanie kodów kreskowych
- przypisywanie lokalizacji magazynowych
- dodawanie zdjęć i opisów technicznych
- przypisanie części do konkretnych maszyn i linii produkcyjnych

## Automatyzacja i workflow

- Integracja dodatkowego skanera kodów kreskowych z systemem kierownika
- Eliminacja ręcznego przepisywania identyfikatorów
- Instruktaż i wdrożenie użytkownika do nowego procesu pracy
- Automatyzacja segregacji wiadomości e-mail:
  - foldery,
  - filtry,
  - archiwizacja danych

## Dokumentacja i serwis

- Organizacja obiegu dokumentów i archiwizacji faktur
- Śledzenie przepływu dokumentacji między działami
- Nadzór nad serwisami wózków widłowych
- Kontrola zakresu wykonanych napraw
- Archiwizacja protokołów serwisowych i dokumentacji UDT

## Reklamacje techniczne

- Analiza uszkodzeń dostarczonych pomp
- Argumentacja techniczna wobec dostawcy
- Wyegzekwowanie wymiany towaru na koszt sprzedawcy

# Rozwiązanie / efekt

- Usprawnienie wyszukiwania części i dokumentów
- Skrócenie czasu identyfikacji elementów magazynowych
- Eliminacja części ręcznych operacji dzięki skanerom kodów
- Zwiększenie kontroli nad obiegiem dokumentacji
- Uporządkowanie komunikacji e-mail i archiwizacji danych
- Ograniczenie strat finansowych poprzez skuteczne reklamacje (~500 zł odzyskane dla firmy)
- Zwiększenie kontroli nad procesami serwisowymi
- Samodzielne rozwiązywanie problemów technicznych bez wsparcia działu IT

# Wnioski techniczne

- Brak standaryzacji obiegu informacji zwiększa ryzyko błędów organizacyjnych
- Integracja prostych narzędzi (skaner + system magazynowy) znacząco zwiększa efektywność pracy
- Automatyczna segregacja danych ogranicza ryzyko utraty informacji
- Dokumentowanie przepływu operacji zwiększa odporność procesów na błędy ludzkie
- Samodzielna analiza problemów i szybkie wdrażanie usprawnień ogranicza przestoje operacyjne
- Wózki widłowe pracujące poza środowiskiem nominalnym ulegają przyspieszonemu zużyciu
